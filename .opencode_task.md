7. test: speed correctness matrix
   - New file: `tests/test_speed.py`.
   - Tests MUST be fast (use short audio, preferably synthetic) and MUST
     not require the MLX model if avoidable. Use `_apply_speed` directly
     on a synthetic sine wave or silence where possible. For end-to-end
     tests that require generation, mark `@pytest.mark.slow` so CI can
     skip them. Default `pytest` run must stay under 5s overhead.
   - Cover at minimum:
     * boundary: 0.5 and 2.0 accepted; 0.499 and 2.001 rejected.
     * ratio tests on synthetic waveform: len(apply_speed(x,0.5)) ≈ 2*len(x),

       ±few samples accurate).

       centroid diff <10% between speed=1.0 and speed=0.5.
     * CLI: `spanish-tts say --speed 3.0 foo` exits non-zero WITHOUT
       loading any model (use click.CliRunner).
     * MCP say: out-of-range returns `{"error": "speed out of range..."}`.
     * graceful fallback: monkeypatch librosa import to fail;
       `_apply_speed` returns input unchanged and logs warning.
